Object Oriented Programming

What is OOP?

OOP, or Object oriented programming refers to a type of computer programming technique that includes or relies on the concept of classes and objects. It is used to structure a software program into simple, reusable pieces of code blueprints (usually called classes) which are used to create individual instances of objects. Each object has its own characteristics (called variables), and can perform actions (called methods) to modify those characteristics. A majority of the world's most popular coding languages are Object Oriented, including:

  • Java
  • C++
  • Python
  • JavaScript
  • Java


Here is an example of a Car class in Java:

/**
 * Car Class which stores the information about the car
 * including the brand, model, mileage, gas level, and tank capacity
 */

 public class Car {
     // Declaring Global Variables
     public String brand;
     public String model;
     public int kilometers;
     public double gas;
     public double tankCapacity;

     /**
      * Car constructor, creates a new object of Car with the following parameters
      * @param b String Brand name
      * @param m String Car make
      * @param g double How much gas is in the car
      * @param tCap double Tank Capacity
      */
      public Car (String b, String m, double g, double tCap) {
          // Initialising global variables
          brand = b;
          model = m;
          gas = g;
          tankCapacity = tCap;
      }

     /**
      * The drive method updates the car with how many kilometres have been driven,
      * and how much gas was consumed on a drive
      * @param distance How far you are driving the call
      */
      public void drive (double distance) {
          kilometers += distance; // Updating distance
          gas -= distance / 5; // Updating how much gas the car consumes
      }

     /**
      * The gasUp method sets the gas level to a full tank
      */
      public void gasUp() {
          // Setting the gas level to the tank capacity
          gas = tankCapacity;
      }
 } // Car class

 public class Main {
      public static void main (String[] args) {
          Car hondaCivic = new Car("Honda", "Sedan", 15.0, 30.0);
          hondaCivic.drive(19.0);
          System.out.println("Kilometres: " + hondaCivic); // Outputs 19.0 km
      }
 }

Why Use OOP?

Simply put, OOP can make programming much more effecient, through code re-usability and inheritence. Inheritence allows a class to pass all of its values, and methods to another class. A Super Class is a class that gets inherited by another Sub Class, and the sub class can add, or change its own methods and variables. Using this method, it allows us to re-use the same code, instead of having to re-write it with subtle changes. Here is an example using Cats and Dogs:

 public class Pet {
      public String name;
      public int age;
      public Pet (String n, int a) {
          name = n;
          age = a;
      }
      public String speak () {
          return name + " said: ";
      }
 }

 public class Dog extends {
      public Pet (String n, int a) {
          super(n, a);
      }
      public String speak () {
          return super.speak() + "woof!";
      }
 }

 public class Cat extends {
      public Pet (String n, int a) {
          super(n, a);
      }
      public String speak () {
          return super.speak() + "meow!";
      }
 }


In this case, instead of rewriting all the variables from the Pet class in the Dog and Cat class, we simply make the subclasses of Pet. The super() call in the constructor allows us to call the constructor from the superclass. Then, in the methods we want to overwrite, or make any changes to we can once again call super.speak(), then add on our additions. Now if we wanted to make a change to the speak() method, all we need to do is edit it in the parent class. Inheritance is a great feature in Object Oriented Programming, which can save alot of time when working with many classes.

What is Encapsulation?

Encapsulation, or data hiding, is method of protecting your local variables in a class. But before we cover Encapsultaion, we have to talk about variable and method access. So far in this tutorial, we've only used public methods and variables, but there are actually four different Access Modifiers. They are:

  1. Default (none)
  2. Public
  3. Private
  4. Protected


These modifers change what variable can be seen, and changed from different classes. Here is a full table of what each Acess Modifier does:

Although there are certain exceptions, we typically want to make most of our Class variables private. This is done for security reasons, so that both the user and anyone with some malicious intent can't access the data that is stored inside of a class. If we were to set every value to be public, then anyone would be able to see, and change the values of our class. However, this presents us with a problem as sometimes we also need to be able to change a class' values from outside of it. They way we get around this is through Accessor and Mutator Methods. which allow us to safely access, and change variables. Here is the Pet class re-written to use encapsulation.

 public class Pet {
      private String name;
      private int age;
      public Pet (String n, int a) {
          name = n;
          age = a;
      }
      public String getName () {
          return name;
      }
      public void setName (String n) {
          name = n;
      }
      public int getAge () {
          return age;
      }
      public void setAge (int a) {
          age = a;
      }
 }
